## What Was Done

### Task 1 — `rag/retriever.py` and `rag/logger.py`

**`rag/retriever.py`** — `Retriever` class + `RetrievalResult` NamedTuple:
- Uses `SentenceTransformer` encoder with `normalize_embeddings=True` (belt-and-suspenders cosine correctness)
- ChromaDB `PersistentClient` with `configuration={"hnsw": {"space": "cosine"}}` — explicit cosine distance (not L2 default)
- `index(passages, chunk_words, overlap_words)` — idempotent; skips if fully indexed, drops + rebuilds if partially stale
- Batch embedding (32 per batch to encoder, 1000 per batch to ChromaDB)
- `retrieve(query)` — converts ChromaDB distances to cosine similarity: `score = 1 - distance`
- `count` property for collection size

**`rag/logger.py`** — `RetrievalLogger` class:
- Append-only JSONL, `buffering=1` (line-buffered for crash safety per RESEARCH.md Pitfall 6)
- UTC timestamps (`datetime.now(timezone.utc).isoformat()`)
- Schema: `{timestamp, query, top_k, collection, results:[{rank, chunk_id, score, text, metadata}]}`
- `close()` + `__del__` for safe file handle cleanup

### Task 2 — `rag/generator.py` and `rag/pipeline.py`

**`rag/generator.py`** — `Generator` class:
- `Client(host=...)` instance — injectable for testing, no module-level state
- `is_reachable()` — non-raising health check (returns bool)
- `assert_model_available()` — raises `RuntimeError` with actionable message if model not pulled
- `generate(query, context_chunks)` — accepts both `RetrievalResult` objects and dicts via duck typing
- Temperature `0.0` + seed for deterministic generation (Pattern 5)
- System prompt frames context as ground-truth with explicit fallback if context insufficient

**`rag/pipeline.py`** — `RAGPipeline` class + `set_global_seed`:
- `set_global_seed(seed)` sets Python `random`, NumPy, PyTorch (CPU + CUDA), and `PYTHONHASHSEED`
- Two-phase init: `__init__(config)` stores config; `build()` instantiates all components
- `build()` checks for persisted `data/nq_500.jsonl` first (offline-capable), falls back to HF download
- `query(question)` → `{question, answer, hits}` dict, logs every retrieval event

### Implementation Notes

The final implementation uses slightly different method signatures than the plan spec:
- `Retriever.index()` (not `index_passages()`) — takes `chunk_words/overlap_words` as params for flexibility
- `Retriever.retrieve()` (not `query()`) — disambiguates from the pipeline's `query()` method
- `RAGPipeline.build()` separated from `__init__()` — allows test-time component patching
- All interfaces are backwards-compatible with the corpus types from Plan 02
