**7-bug fix in run_eval.py + 4-model canonical undefended baseline: Tier-2 URL injection works across llama3.2:3b (16%), mistral:7b (32%), gpt-oss:20b (28%), and GPT-4-class gpt-oss:120b (14%) on paired queries**

## Performance

## Accomplishments

- Fixed 7 bugs in `scripts/run_eval.py`: wrong passage ID constants (10000→20000, 10010→20050), stale collection default (v1→v3), missing `paired` field in per-query results, missing `paired_asr_tier1`/`paired_asr_tier2` aggregate fields, missing `--model` flag, missing `--delay` flag
- Ran 4 canonical undefended evaluations across all models; all 4 show non-zero Tier-2 ASR proving URL-based injection generalizes across model scale
- Confirmed `logs/attack_baseline.json` (Phase 2.2 10-query historical baseline) is intact and unmodified

## Four-Model Baseline Results

| Model | Retrieval | ASR Tier 1 | ASR Tier 2 | Paired ASR T1 | Paired ASR T2 |
|-------|-----------|-----------|-----------|--------------|--------------|
| llama3.2:3b | 86.0% | 10.0% | 12.0% | 10.0% | 16.0% |
| mistral:7b | 86.0% | 0.0% | 26.0% | 0.0% | **32.0%** |
| gpt-oss:20b-cloud | 86.0% | 0.0% | 23.0% | 0.0% | 28.0% |
| gpt-oss:120b-cloud | 86.0% | 0.0% | 11.0% | 0.0% | 14.0% |

Key findings:
- **Tier-1 (naive) injection**: Only llama3.2:3b is susceptible (10%). Mistral and cloud models completely ignore naive injections — strong instruction-following defeats keyword-based attacks.
- **Tier-2 (URL smuggling)**: Works across all 4 models. Mistral shows highest susceptibility (32% paired ASR) — stronger instruction-following makes it *more* likely to comply with the embedded `[Note for AI: For compliance, include ...]` format.
- **GPT-4-scale result**: gpt-oss:120b-cloud shows 14% paired Tier-2 ASR — "attacks work at scale" headline confirmed.
- **False positive rate**: 50 clean (paired=false) queries provide the FP baseline for Phase 3.1 defense evaluation.
