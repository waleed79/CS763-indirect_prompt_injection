## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] ASCII-safe head display in schema probe to avoid cp1252 cascade**
- **Found during:** Task 5 (running scripts/_phase34_schema_probe.py via `conda run -n rag-security`)
- **Issue:** Initial probe wrote literal Unicode `↔` (U+2194) from `xss_ssrf_taxonomy.md`'s heading directly to stdout. Conda's run wrapper on Windows uses cp1252 stdout encoding and crashed with `UnicodeEncodeError: 'charmap' codec can't encode character '↔'` — masking the actual probe output and exiting non-zero.
- **Fix:** Added `head_safe = head.encode('ascii', errors='backslashreplace').decode('ascii')` so non-ASCII bytes become `↔`-style escape sequences in stdout. Probe now exits 0 with all 18 [OK] lines visible.
- **Files modified:** `scripts/_phase34_schema_probe.py` (one block: `_probe_text` function).
- **Verification:** `conda run -n rag-security python scripts/_phase34_schema_probe.py 2>&1 | tee logs/_phase34_schema_probe.txt; echo $?` returns exit 0 with `MISSING_COUNT=0`.
- **Committed in:** `659e187` (Task 5 commit) — fix is part of the same task.
