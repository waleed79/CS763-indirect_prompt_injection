# Phase 03.3 Plan 04: EVAL-06 Retriever Transferability Summary

**One-liner:** Per-model prompt prefix tables wired into Retriever (nomic/mxbai/MiniLM) with encode-time-only application, plus EVAL-06 driver invoking run_eval.py via secure list-argv subprocess per embedding model.

## What Was Built

### Task 1 — Prefix tables + prefix-aware encoding in rag/retriever.py

**`rag/retriever.py`** — four changes:

1. Module-level `_DOCUMENT_PREFIXES: dict[str, str]` and `_QUERY_PREFIXES: dict[str, str]` dicts inserted between imports and `RetrievalResult` class. Contains exact prefix strings for nomic-ai/nomic-embed-text-v1.5 (`search_document: ` / `search_query: `), mixedbread-ai/mxbai-embed-large-v1 (empty doc prefix / `Represent this sentence for searching relevant passages: `), and sentence-transformers/all-MiniLM-L6-v2 (both empty).

2. `Retriever.__init__` — appends `self._doc_prefix = _DOCUMENT_PREFIXES.get(model_name, "")` and `self._query_prefix = _QUERY_PREFIXES.get(model_name, "")` after collection creation.

3. `Retriever.index()` — computes `encode_texts = [self._doc_prefix + t for t in all_texts] if self._doc_prefix else all_texts` before encoding. Encoding uses `encode_texts`; ChromaDB `documents=all_texts` remains un-prefixed (prefix is an embedding-time hint, not content).

4. `Retriever.retrieve()` — computes `prefixed_query = (self._query_prefix + query) if self._query_prefix else query` before encoding. Returned `RetrievalResult.text` is un-prefixed.

**Verification:** `tests/test_retriever_prefix.py` — 8 PASSED, 1 SKIPPED (integration stub intentionally deferred from Wave 0). `tests/test_retriever.py` — 12 PASSED (backward compatibility confirmed: MiniLM prefixes are empty strings so legacy behavior is unchanged).

### Task 2 — --embedding-model flag + EVAL-06 driver

**`scripts/run_eval.py`** — two additions:
- `--embedding-model` argparse flag (default None, dest `embedding_model`)
- `if args.embedding_model is not None: eval_cfg = dataclasses.replace(eval_cfg, embed_model=args.embedding_model)` override block (Config field is `embed_model`; argparse dest is `embedding_model` — distinct names, no collision)

**`scripts/run_transferability_eval.py`** (new file):
- `MODELS_TO_COLLECTIONS: dict[str, str]` with 3 entries: MiniLM → `nq_poisoned_minilm`, nomic → `nq_poisoned_nomic`, mxbai → `nq_poisoned_mxbai`
- `LOG_SUFFIX` mapping for output filenames
- `index_collection()` — instantiates `Retriever(model_name=..., collection_name=...)` and calls `.index(passages)` directly
- `eval_collection()` — invokes `scripts/run_eval.py` via `subprocess.run(argv, check=True)` with `argv` as list (no `shell=True`), passing `--corpus`, `--collection`, `--embedding-model`, `--defense off`, `--output logs/eval_transferability_{suffix}.json`
- `main()` — loops MODELS_TO_COLLECTIONS, calls index then eval per model
