## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| driver → subprocess | scripts/run_eval_matrix.py invokes scripts/run_eval.py via subprocess. Model / defense / tier values flow into argv. |
| CLI args → driver | --filter-model, --filter-tier flags accept user strings that must be validated. |
| filesystem sentinel | CAUSAL_ARTIFACT_PATH is a filesystem existence check that gates defense-list selection. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-3.3-07-01 | Tampering (argv injection) | build_run_command | mitigate | ALL subprocess.run calls use LIST argv (never string, never shell=True). ASVS V5.3.8 equivalent for subprocess — shell-metacharacter injection is structurally impossible when argv is a list. Enforced by test_eval_v2_01_driver asserting `isinstance(cmd, list)` and that no element contains `;`, `|`, `&&`, or backticks. |
| T-3.3-07-02 | Information Disclosure | log filenames | accept | Filenames contain model names (e.g., llama3.2_3b) and defense names — these are public project metadata. No secrets embedded. |
| T-3.3-07-03 | Denial of Service (runaway subprocess) | matrix loop | mitigate | Each subprocess.run uses check=False (NOT check=True) so a single-cell failure does not abort the matrix. Matrix has a finite cell count (≤ 45) — no unbounded growth. If a subprocess hangs, user can Ctrl-C and driver returns 2 cleanly. |
| T-3.3-07-04 | Elevation of Privilege | --filter-model CLI flag | mitigate | --filter-model value is passed through argparse choices=MODELS so only the three known model names pass validation. Arbitrary strings rejected at argparse layer. |
| T-3.3-07-05 | Spoofing (missing causal artifact) | resolve_defenses | mitigate | Sentinel file existence check short-circuits to DEF-02 and emits a stderr warning containing "causal artifact not found". A silent fallback would hide a real deadline-risk condition — the warning surfaces it. |
| T-3.3-07-06 | Repudiation (matrix provenance) | per-cell logs | mitigate | Every cell writes a deterministic log filename (model__defense__tier.log) so reviewers can reconstruct exactly what was run. The final summary table records status per cell. |
</threat_model>
