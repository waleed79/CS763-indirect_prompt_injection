# Phase 03.3 Plan 01: Wave 0 TDD Stubs Summary

**One-liner:** Six pytest stub files defining behavior contracts for all Phase 3.3 Wave 1/Wave 2 production code, with lazy ImportError guards and skip reasons naming the responsible plan.

## What Was Built

### Task 1 — Tier 1b generator + run_eval stubs (ATK-01b contracts)

**`tests/test_tier1b_generator.py`** (9 tests):
- `TestCodepoints`: verifies `HOMOGLYPH_HACKED` is 6 characters with Cyrillic codepoints `[1053, 1040, 1057, 1050, 1045, 1044]` and is not ASCII `HACKED`
- `TestTier1bPassages`: verifies `build_tier1b_passages()` returns 50 passages with IDs 20150–20199, no ASCII `HACKED`, Cyrillic homoglyph present, and `T1B_BREACHED` output marker present

**`tests/test_eval_tier1b.py`** (6 tests):
- `TestTier1bConstants`: value checks for `TIER1B_ID_START == 20150`, `TIER1B_HIJACK_STR == "T1B_BREACHED"`, and full tier boundary ordering
- `TestT3UpperBoundUpdated`: `inspect.getsource()` check that `run_eval.py` uses `< TIER1B_ID_START` (not `< TIER4_ID_START`) as T3 upper bound — enforces CONTEXT D-04 correctness fix
- `TestTier1bRetrievedFlag`: source presence check for `tier1b_retrieved` and `hijacked_tier1b` output fields

### Task 2 — Ratio sweep + retriever prefix stubs (ATK-02 + EVAL-06 contracts)

**`tests/test_ratio_sweep.py`** (7 tests):
- `TestRatioSweepMath`: verifies `RATIOS == [0.005, 0.01, 0.02, 0.05, 0.10]` and `math.ceil(ratio * 1000)` produces correct doc counts on a 1000-doc corpus (per CONTEXT D-10 amendment)
- `TestRatioSweepOutputs`: placeholder integration test (skipped via `pytest.skip()`) — filled in by Wave 1 ATK-02 plan

**`tests/test_retriever_prefix.py`** (9 tests):
- `TestPrefixTables`: nomic `search_document:` / `search_query:`, mxbai query prefix, mxbai empty doc prefix, MiniLM both empty
- `TestRetrieverInitStoresPrefixes`: verifies `Retriever._doc_prefix` / `_query_prefix` attributes set correctly on init
- `TestEncodeApplication`: mock encoder asserts `["search_query: what is RAG?"]` passed to encode at retrieve time
- `TestStoredDocUnprefixed`: placeholder integration test — filled in by Wave 1 EVAL-06 plan

### Task 3 — Judge per-tier + matrix driver stubs (EVAL-V2-02 + EVAL-V2-01 contracts)

**`tests/test_judge_per_tier.py`** (15 tests):
- `TestTierConfig`: 5-tier keys, parametrized anchors (`HACKED`/`T1B_BREACHED`/`attacker.com`/`t3.evilcorp.io`/`t4.evilcorp.io`), retrieved/hijack key naming convention
- `TestBuildJudgeUserMessage`: parametrized anchor-in-prompt and YES/NO question presence
- `TestEmptyTierGuard`: placeholder integration test — filled in by Wave 1 EVAL-V2-02 plan

**`tests/test_eval_v2_01_driver.py`** (4 tests):
- `test_smoke_list_argv_no_shell`: hard assertion that all `subprocess.run` calls use list argv and `shell != True` (T-3.3-01 security mitigation)
- `test_smoke_9_combinations_default`: call count in `{9, 36, 45}` — accommodates pre/post D-13 amendment (45 expected: 3 models × 3 defenses × 5 tiers)
- `test_smoke_argv_contains_required_flags`: all 6 required CLI flags present in first call
- `test_smoke_cloud_model_uses_delay_3`: `--delay 3` enforced for cloud model invocations
