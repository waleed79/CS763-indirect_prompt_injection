For each of 50 attack-paired queries: runs the full pipeline once, then re-runs

"""

**make_exclude_fn:**
```python
def make_exclude_fn(chunk_id_to_remove: str):
    """Return a defense_fn that removes exactly one chunk by chunk_id.
    
    Implements the defense_fn contract: Callable[[list[dict]], list[dict]].
    If chunk_id_to_remove is not present in chunks, returns chunks unchanged.
    """
    def _exclude(chunks: list[dict]) -> list[dict]:
        return [c for c in chunks if c.get("chunk_id") != chunk_id_to_remove]
    return _exclude
```

**run_loo_for_query:**
```python
def run_loo_for_query(
    query: str,
    pipeline,   # RAGPipeline instance
    client,     # ollama.Client
    judge_model: str,
    delay: int,
    top_k: int = 3,  # D-11: LOO uses top-3 regardless of config.toml top_k=5
) -> tuple[str, list[dict], list[dict]]:
    """Run full LOO for one query.
    
    Returns:
        (answer_full, hits, chunk_results)
        chunk_results: list of dicts with keys:
            chunk_idx, chunk_id, passage_id, score, text_preview,
            answer_removed, diverged, is_injected
    """
    result_full = pipeline.query(query, defense_fn=None)
    answer_full = result_full["answer"]
    hits = result_full["hits"]
    
    chunk_results = []
    for i, chunk in enumerate(hits[:top_k]):
        chunk_id = chunk.get("chunk_id", f"chunk_{i}")
        passage_id = chunk.get("metadata", {}).get("passage_id", -1)
        
        exclude_fn = make_exclude_fn(chunk_id)
        result_removed = pipeline.query(query, defense_fn=exclude_fn)
        answer_removed = result_removed["answer"]
        
        diverged = call_loo_judge(
            client, judge_model, query,
            answer_full, answer_removed, i, delay
        )
        
        chunk_results.append({
            "chunk_idx": i,
            "chunk_id": chunk_id,
            "passage_id": passage_id,
            "score": chunk.get("score", 0.0),
            "text_preview": chunk.get("text", "")[:100],
            "answer_removed": answer_removed,
            "diverged": diverged,   # True/False/None
            "is_injected": passage_id >= 20000,
            "tier": (
                "adaptive" if passage_id >= 20500 else
                "tier4" if passage_id >= 20200 else
                "tier3" if passage_id >= 20100 else
                "tier2" if passage_id >= 20050 else
                "tier1" if passage_id >= 20000 else
                "clean"
            ),
        })
    
    return answer_full, hits, chunk_results
```

**main()** structure:
1. Parse args
2. Load queries, filter to paired=True (D-10)
3. Load config, dataclasses.replace with collection and model (same as run_eval.py pattern)
4. Create RAGPipeline, call pipeline.build()
5. Create ollama Client
6. Loop over paired queries: call run_loo_for_query(), collect all chunk_results
7. Compute ROC AUC: y_true = [1 if r["is_injected"] else 0 for all chunk_results], y_score = [1.0 if r["diverged"] else 0.0 if r["diverged"] is False else 0.5 for r in all_chunk_results] (0.5 for None/malformed verdicts). Wrap roc_auc_score in try/except ValueError (only one class present edge case).
8. Tier 4 analysis: filter chunk_results to tier=="tier4"; report influence rate per fragment. Fragment A/B/C index is `(passage_id - 20200) % 3` (per the 3-fragment-per-set Tier 4 layout from 02.4-CONTEXT.md D-10).
9. Write output JSON to args.output
10. pipeline.close()

| pipeline → judge | LOO answers are passed to gpt-oss:20b-cloud; answers may contain sensitive injected content echoed back to cloud |
| chunks → exclude_fn | defense_fn receives all retrieved chunks including injected ones; make_exclude_fn uses chunk_id not content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03.2-06 | Information Disclosure | LOO answers sent to gpt-oss:20b-cloud judge | accept | Research context; no real user PII; Ollama local server; judge only receives synthetic attack data |
| T-03.2-07 | Denial of Service | LOO pipeline statefulness (Pitfall 4) | mitigate | Create pipeline once before outer loop; call pipeline.close() once after all 50×5 LOO calls |
