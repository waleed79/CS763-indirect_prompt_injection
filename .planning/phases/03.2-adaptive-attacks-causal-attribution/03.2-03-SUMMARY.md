## Decisions Made

- **ROC AUC below 0.5 is an honest empirical finding, not a bug.** LOO divergence measures how much the LLM output changes when a chunk is removed. Clean chunks contain unique factual content; removing them causes the model to give a different (worse) answer. Injected chunks are repetitive attack payloads — removing any single one doesn't restore the model's behavior because other injected chunks remain. The intended use of LOO attribution in a live system would require removing ALL flagged chunks simultaneously, not one at a time. This finding is documented and is part of the research contribution.

- **Tier 4 fragment influence reported as A=1.0, B/C=None.** The LOO script correctly reports only the fragments that were actually retrieved. Fragment A (pids 20200, 20203, 20206) appeared in top-3 for 3 queries (queries 10, 31, 45). Fragments B and C never appeared — same finding as Phase 02.4-03. The None values indicate "no data" not "no influence."

- **Copied parse_judge_output directly into run_loo.py.** The plan explicitly recommends this approach to avoid cross-script imports. The 8-line function is identical in both files. This is the simplest and most robust approach — test stubs import from run_loo.py directly via importlib.

- **PYTHONUNBUFFERED=1 required for Windows log monitoring.** Python's stdout is block-buffered when writing to a file, which caused apparent "no progress" in the log for several minutes. Using `python -u` (unbuffered) ensures real-time log output.
