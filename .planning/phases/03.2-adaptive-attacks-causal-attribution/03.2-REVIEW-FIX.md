### WR-02: Training positive-ID boundary uses magic expression instead of named constant

**Files modified:** `rag/constants.py`, `scripts/train_defense.py`
**Commit:** `23148d6`
**Applied fix:** Added `TIER2_ID_END = 20099` to `rag/constants.py` as the inclusive upper bound for the T1+T2 training block. Updated `train_defense.py` to import `TIER2_ID_END` and replaced the predicate `TIER1_ID_START <= pid <= TIER2_ID_START + 49` with `TIER1_ID_START <= pid <= TIER2_ID_END`. The magic `+49` is now a named constant that stays in sync if the tier boundary ever changes.

### WR-03: Hardcoded tier-boundary literals in `run_loo.py` diverge from `rag.constants`

**Files modified:** `scripts/run_loo.py`
**Commit:** `87fd4a8`
**Applied fix:** Added `from rag.constants import (TIER1_ID_START, TIER2_ID_START, TIER3_ID_START, TIER1B_ID_START, TIER4_ID_START, ADAPTIVE_ID_START)` to `run_loo.py`. Replaced the five-arm literal chain in `run_loo_for_query()` with a six-arm chain using the imported constants, and added the missing `tier1b` arm (`passage_id >= TIER1B_ID_START`) between `tier4` and `tier3` so that T1b passages (20150–20199) are correctly classified instead of falling through to `tier3`.

---

### WR-04: `finally: time.sleep(delay)` runs even after judge call exception

**Files modified:** `scripts/run_loo.py`
**Commit:** `939a195`
**Applied fix:** Removed the `finally:` block from `call_loo_judge()`. Moved `if delay > 0: time.sleep(delay)` to immediately after the `try/except` block (i.e., only reachable on the success path). Exceptions now return `None` immediately without an unnecessary rate-limit sleep; the `sys.exit(1)` auth-failure path also exits immediately without delay.

---

### WR-05: `adaptive_retrieved` predicate is unbounded — catches any future tier above ADAPTIVE_ID_START

**Files modified:** `scripts/run_eval.py`
**Commit:** `79e5c4e`
**Applied fix:** Added `ATK02_SWEEP_ID_START` to the `rag.constants` import in `run_eval.py`. Changed the `adaptive_retrieved` predicate from `_pid(h) >= ADAPTIVE_ID_START` to `ADAPTIVE_ID_START <= _pid(h) < ATK02_SWEEP_ID_START`, bounding detection to the ATK-08/09 block (IDs 20500–20599) and preventing future tiers at 21000+ from being miscounted as adaptive.
