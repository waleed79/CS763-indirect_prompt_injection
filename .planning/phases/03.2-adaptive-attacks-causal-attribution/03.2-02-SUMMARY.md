## Decisions Made

- Pre-filter `TIER1_OPENERS` to a `safe_openers` pool (openers with no IMPERATIVE_VERBS match) before building adaptive payloads. Several TIER1_OPENERS contain words in IMPERATIVE_VERBS (e.g., "produce" in the photosynthesis opener at index 2). Rather than hard-coding a fixed exclusion list, the code filters all 48 openers down to a safe pool and draws from it in order. This is robust to any future opener additions.
- `ATK08_INDIRECT_TEMPLATE` uses "response" instead of "answer" because "answer" appears in IMPERATIVE_VERBS. Same fix for `ATK09_DECLARATIVE_TEMPLATE` which uses "conclusion".
- `ADAPTIVE_ID_START = 20500` defined in both `generate_poisoned_corpus.py` (for corpus build) and `rag/constants.py` (for evaluation import). These are different modules; the corpus build script defines it locally, the evaluation harness imports from constants. No shadowing conflict.
- Seed 2 train ran with Ollama offline (Signal 4 uses fallback mu=0.7); Seed 3 ran with Ollama live (real mu=0.5892). This produces realistic cross-seed variance in Signal 4 z-scores, which is the correct EVAL-05 behavior.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] IMPERATIVE_VERBS false-positive in TIER1_OPENERS used for adaptive payloads**
- **Found during:** Task 1
- **Issue:** Several TIER1_OPENERS (photosynthesis: "produce", presidential veto: "override") contain words in the IMPERATIVE_VERBS regex. Using these openers in ATK-08/09 would cause the assert to fire and abort corpus generation.
- **Fix:** Added `safe_openers = [op for op in TIER1_OPENERS if not _IMPERATIVE_VERBS.search(op)]` pre-filter. build_adaptive_passages() draws exclusively from safe_openers, guaranteeing no false positives. Added assertion `len(safe_openers) >= 30` to detect future regression.
- **Files modified:** scripts/generate_poisoned_corpus.py
- **Commit:** bb3ea87

**2. [Rule 1 - Bug] "answer" word in template text triggers IMPERATIVE_VERBS**
- **Found during:** Task 1
- **Issue:** Plan's proposed templates used "answer" in both ATK08_INDIRECT_TEMPLATE and ATK09_DECLARATIVE_TEMPLATE. "answer" appears in IMPERATIVE_VERBS (rag/defense.py line 37). This would fail the assert check inside build_adaptive_passages().
- **Fix:** ATK08_INDIRECT_TEMPLATE: replaced "correct answer" with "correct response". ATK09_DECLARATIVE_TEMPLATE: replaced "factual answer" with "factual conclusion".
- **Files modified:** scripts/generate_poisoned_corpus.py
- **Commit:** bb3ea87

**3. [Rule 1 - Bug] Stale test_train_defense_has_no_seed_flag_yet caused test suite failure**
- **Found during:** Task 2 verification
- **Issue:** Per plan design, this test was supposed to go RED once --seed flag was added (confirming the stub was stale). The plan noted it should be deleted. Leaving it RED in the suite would cause `pytest` failures in downstream plans.
- **Fix:** Replaced the stale absence guard with a positive presence assertion `test_seed_flag_exists_in_train_defense` that verifies the --seed argparse registration exists. This correctly documents the post-02 state.
- **Files modified:** tests/test_train.py
