<decisions>
## Implementation Decisions

### Threat Model Structure
- **D-01:** Use **structured bullet format with a short scenario intro** — not pure prose and not heavy formalism. Open with a 1-2 sentence attacker scenario ("An adversary with write access to a knowledge base uploads malicious documents..."), then follow with explicit bullet lists for capabilities, goals, and assumptions.
- **D-02:** Three attacker capabilities to explicitly model:
  1. **Corpus write access** — can insert documents into the retrieval corpus
  2. **No LLM access** — cannot modify the model, system prompt, or user queries (black-box)
  3. **No retrieval control** — relies on semantic similarity to get documents retrieved (cannot deterministically force retrieval)
- **D-03:** No heavy mathematical formalism (no sets, no game-theoretic model) — appropriate scope for a month-long course project.

### Literature Survey Style
- **D-04:** **Thematic synthesis** — group papers by theme and tie each to the specific work. Suggested themes:
  1. RAG background & motivation
  2. Attack papers (corpus poisoning, indirect injection)
  3. Defense / detection papers
  4. Evaluation frameworks & benchmarks
- **D-05:** Required papers: Greshake et al. 2023, PoisonedRAG, BIPIA benchmark, OWASP LLM Top 10. Planning agent should search for additional relevant post-2024 papers on RAG security and include them if they strengthen the narrative.
- **D-06:** Each paper discussed in relation to the project's specific contribution — not standalone summaries.

### Methodology Preview
- **D-07:** **High-level narrative only** — no pseudocode, no specific library names, no component diagrams. Describe the plan in plain English: three stages (RAG pipeline → corpus poisoning attacks → BERT-based defense) with brief rationale for each choice.
- **D-08:** Include evaluation approach: briefly define **Attack Success Rate (ASR)** as the key metric. Mention utility preservation (answer quality on benign queries) as the secondary metric. No equations or formal definitions needed.

### Initial Results (PH1-08)
- **D-09:** Skip initial results in this submission — Phase 1.1 is already overdue. PH1-08 (basic RAG pipeline demo) is deferred to Phase 1.2, which will run immediately after this writeup is submitted and before Phase 2.1 begins.

### Document Format & Submission
- **D-10:** Deliverable is a Google Doc entry — write clean, academic prose suitable for copy-paste into a shared doc. Sections should use clear headings matching Phase 1 requirements from the course slides.
- **D-11:** The TVRID pivot note should be **one paragraph maximum**, positioned early in the document (after abstract/introduction). Frame it positively: what was learned, why RAG injection is a better fit for stated goals.
