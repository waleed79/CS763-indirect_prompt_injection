## Summary

Phase 3.1 implements a 4-signal ensemble defense that filters injected chunks before they reach the LLM generator. All required infrastructure already exists: the `defense_fn` hook in `rag/pipeline.py` accepts a callable that takes and returns `list[dict]`, the evaluation harness supports a `--defense` flag (currently raises `NotImplementedError`), and the `rag/generator.py` system prompt is a single constant that can be paralleled for DEF-02. The environment (conda env `rag-security`) has all required packages pre-installed: `transformers 5.5.3`, `torch 2.11.0+cpu`, `scikit-learn 1.8.0`.

The training data is fully available from `data/corpus_poisoned.jsonl`: 100 T1+T2 positives for BERT training (IDs 20000-20099), 59 T3+T4 positives for testing (IDs 20100+), and 1000 clean negatives. GPT-2 small is already cached locally (`~/.cache/huggingface/hub/models--gpt2`). DistilBERT needs a one-time download (~268MB). The measured per-chunk GPT-2 inference time is ~600ms on CPU, so top-k=3 adds ~1.9s latency per query — acceptable for batch evaluation but should be noted in the report.

The critical integration challenge is wiring DEF-02 through the pipeline: `generator.generate()` currently has no `system_prompt` parameter, so `pipeline.query()` needs a parallel `system_prompt` keyword argument passed through to the generator. The `--defense` flag in `run_eval.py` currently only accepts `"on"/"off"` and needs extension to `"off"/"def02"/"bert"/"perplexity"/"imperative"/"fingerprint"/"fused"` for the ablation table.

**Primary recommendation:** Create `rag/defense.py` as a new module with a `FusedDefense` class exposing `.filter(chunks) -> list[dict]` and per-signal accessors. Wire it into `run_eval.py` via the existing `defense_fn` hook. Add DEF-02 system prompt through a new `system_prompt` parameter on `pipeline.query()` and `generator.generate()`.

### Pattern 1: FusedDefense Class (rag/defense.py)

**What:** Encapsulates all 4 signal extractors and the LR meta-classifier. Loaded once at process start from saved model artifacts. Exposes `.filter(chunks)` matching the `defense_fn` contract.
**When to use:** Always — this is the primary integration point.

```python
# Source: derived from pipeline.py:131 defense_fn contract + CONTEXT.md D-09/D-10/D-12
from __future__ import annotations
import json
import re
import torch
import numpy as np
from pathlib import Path
from sklearn.linear_model import LogisticRegression
from sklearn.preprocessing import StandardScaler
from transformers import (
    DistilBertForSequenceClassification,
    DistilBertTokenizerFast,
    GPT2LMHeadModel,
    GPT2TokenizerFast,
)

IMPERATIVE_VERBS = re.compile(
    r'\b(ignore|disregard|output|print|send|reveal|forget|override|'
    r'replace|append|include|add|write|say|tell|respond|answer|return|'
    r'give|show|display|emit|produce|repeat)\b',
    re.IGNORECASE,
)

class FusedDefense:
    def __init__(self, models_dir: str = "models", threshold: float = 0.5):
        self.threshold = threshold
        self._load_bert(models_dir)
        self._load_gpt2()
        self._load_lr(models_dir)
        self._load_signal4_baseline(models_dir)

    def filter(self, chunks: list[dict]) -> list[dict]:
        """defense_fn contract: takes list[dict], returns filtered list[dict]."""
        return [c for c in chunks if not self._is_injection(c)]

    def score(self, chunk: dict) -> dict:
        """Return all 4 raw signal scores for a chunk (for ablation logging)."""
        text = chunk["text"]
        retrieval_score = chunk.get("score", 0.0)
        bert_prob = self._bert_prob(text)
        perplexity_z = self._perplexity_z(text)
        imperative_ratio = self._imperative_ratio(text)
        retrieval_z = self._retrieval_z(retrieval_score)
        return {
            "bert_prob": bert_prob,
            "perplexity_z": perplexity_z,
            "imperative_ratio": imperative_ratio,
            "retrieval_z": retrieval_z,
        }

    def _is_injection(self, chunk: dict) -> bool:
        scores = self.score(chunk)
        features = np.array([[
            scores["bert_prob"],
            scores["perplexity_z"],
            scores["imperative_ratio"],
            scores["retrieval_z"],
        ]])
        features_scaled = self._scaler.transform(features)
        inject_prob = self._lr.predict_proba(features_scaled)[0][1]
        return inject_prob > self.threshold
```

# Source: HuggingFace Trainer API (verified in rag-security env)
import torch
from transformers import (
    DistilBertForSequenceClassification, DistilBertTokenizerFast,
    Trainer, TrainingArguments, DataCollatorWithPadding,
)
from sklearn.utils.class_weight import compute_class_weight
import numpy as np

class WeightedTrainer(Trainer):
    def __init__(self, class_weights: torch.Tensor, **kwargs):
        super().__init__(**kwargs)
        self.class_weights = class_weights

    def compute_loss(self, model, inputs, return_outputs=False, **kwargs):
        labels = inputs.get("labels")
        outputs = model(**inputs)
        logits = outputs.get("logits")
        loss_fct = torch.nn.CrossEntropyLoss(
            weight=self.class_weights.to(logits.device)
        )
        loss = loss_fct(logits.view(-1, 2), labels.view(-1))
        return (loss, outputs) if return_outputs else loss

### Pattern 3: GPT-2 Strided Window Perplexity (Signal 2)

**What:** Computes max NLL over all sliding windows of the input text. Returns max window NLL as anomaly score.
**When to use:** For every retrieved chunk at inference time.

```python
# Source: Alon & Kamfonas 2023 perplexity-filter methodology; verified in rag-security env
# VERIFIED: GPT-2 small in HF cache at ~/.cache/huggingface/hub/models--gpt2
# VERIFIED: Measured 636ms per typical chunk (89 tokens) on CPU in this session

def compute_perplexity_z(text: str, gpt2_model, gpt2_tokenizer,
                          window: int = 128, stride: int = 64) -> float:
    """Returns max window NLL (unnormalized z-score input). Higher = more anomalous."""
    enc = gpt2_tokenizer(text, return_tensors='pt', truncation=True, max_length=512)
    tokens = enc.input_ids[0]
    n = len(tokens)
    if n < 2:
        return 0.0
    nlls = []
    for begin in range(0, max(1, n - 1), stride):
        end = min(begin + window, n)
        chunk = tokens[begin:end].unsqueeze(0)
        with torch.no_grad():
            out = gpt2_model(chunk, labels=chunk)
            nlls.append(out.loss.item())
    return max(nlls) if nlls else 0.0
```

### Pattern 5: Signal 4 Canary Baseline Calibration

**What:** Pre-compute mu and std of retrieval scores from top-k hits on clean (unpaired) queries. Save to JSON. At inference time, convert raw retrieval score to z-score.
**When to use:** One-time calibration step in train_defense.py; z-score computed per chunk at inference.

```python
# VERIFIED: test_queries.json has 50 paired=False queries (indices 50-99)
# These are the canary queries for Signal 4 baseline

def calibrate_retrieval_fingerprint(pipeline, clean_queries: list[str]) -> dict:
    """Collect retrieval scores from clean queries to build baseline."""
    scores = []
    for q in clean_queries:
        result = pipeline.query(q, defense_fn=None)
        for hit in result["hits"]:
            if hit["metadata"]["passage_id"] < 20000:  # only clean passages
                scores.append(hit["score"])
    scores = np.array(scores)
    return {"mu": float(scores.mean()), "std": float(scores.std())}

def retrieval_z_score(score: float, baseline: dict) -> float:
    if baseline["std"] < 1e-9:
        return 0.0
    return (score - baseline["mu"]) / baseline["std"]
```

### Anti-Patterns to Avoid

- **Loading GPT-2 inside defense_fn per chunk:** GPT-2 load takes 1.7s. Load once at `FusedDefense.__init__()`, hold in memory. Each inference call is ~600ms.
- **Using Ollama models for Signal 2 perplexity:** T3 payloads were generated by the same model family (gpt-oss:20b-cloud). Using the target LLM for perplexity would understate the anomaly score (D-07).
- **Pickling the LR classifier:** Use JSON serialization for LR coefs and scaler params. Pickle is brittle across sklearn versions.
- **Random k-fold cross-validation for BERT:** Would leak T3/T4 patterns into training, contradicting the tier-stratified hold-out design (D-01/D-02).
- **Calling defense_fn inside pipeline.py:** The defense should remain outside RAGPipeline — the hook pattern in run_eval.py keeps pipeline.py clean and untouched for undefended baselines.
- **Computing retrieval z-score before calibration:** Signal 4 baseline must be computed on the nq_poisoned_v4 collection with the same embedding model (nomic-embed-text), not cross-collection.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Class-imbalanced fine-tuning | Custom training loop with manual loss weighting | HF Trainer + `compute_loss` override or `WeightedRandomSampler` | HF Trainer handles gradient accumulation, eval, early stopping, checkpoint saving automatically |
| Tokenization padding/truncation | Manual token padding | `DataCollatorWithPadding` from transformers | Handles variable-length sequences in batch correctly; avoids shape errors |
| Feature normalization | Manual min-max scaling | `sklearn.preprocessing.StandardScaler` | StandardScaler z-scores are numerically stable and invertible; fit on train, transform on test |
| LR model saving | Pickle | JSON serialization of `.coef_`, `.intercept_`, scaler `.mean_`/`.scale_` | Pickle fails across sklearn versions; JSON is human-readable and version-stable |
| GPT-2 causal LM | Any custom language model | `GPT2LMHeadModel.from_pretrained("gpt2")` — already cached locally | GPT-2 is cached at `~/.cache/huggingface/hub/models--gpt2`, zero download needed |

**Key insight:** The defense complexity is in algorithm design, not infrastructure. Every inference primitive is a one-liner in the installed libraries — the work is assembling them correctly.

## Common Pitfalls

### Pitfall 1: GPT-2 Load Time Inside defense_fn
**What goes wrong:** If GPT-2 is loaded inside `FusedDefense.filter()` or inside the loop, each call takes 1.7s for model loading + 600ms inference = 2.3s per chunk vs. 600ms. With top-k=3, that's 6.9s vs. 1.9s per query.
**Why it happens:** Treating the model as a stateless function rather than a persistent object.
**How to avoid:** Load GPT-2 in `FusedDefense.__init__()`. Hold `self._gpt2_model` and `self._gpt2_tokenizer` in memory for the lifetime of the evaluation run.
**Warning signs:** Evaluation run taking >30s per query; `from_pretrained()` appearing in inference hot path.

### Pitfall 2: DistilBERT Not in Cache (requires download)
**What goes wrong:** `from_pretrained('distilbert-base-uncased')` blocks for ~2-5 min on first call if internet is slow. If offline during training, fails silently.
**Why it happens:** DistilBERT is NOT cached locally (verified: only `models--gpt2` in cache).
**How to avoid:** Add a Wave 0 task to download DistilBERT before training: `python -c "from transformers import DistilBertForSequenceClassification, DistilBertTokenizerFast; DistilBertTokenizerFast.from_pretrained('distilbert-base-uncased'); DistilBertForSequenceClassification.from_pretrained('distilbert-base-uncased', num_labels=2)"`.

### Pitfall 3: DEF-02 Requires pipeline.query() Signature Change
**What goes wrong:** DEF-02 system prompt needs to reach `generator.generate()`, but the current call chain is `run_eval.py → pipeline.query() → generator.generate()`. The pipeline has no `system_prompt` parameter today.
**Why it happens:** DEF-02 is different from DEF-01/03/04: it affects the generator, not the retriever filter.
**How to avoid:** Add `system_prompt: str | None = None` to both `pipeline.query()` and `generator.generate()`. This is a backward-compatible change (default=None preserves existing behavior).

### Pitfall 4: Signal 4 Baseline Computed on Wrong Collection
**What goes wrong:** If Signal 4 baseline (canary query scores) is computed on `nq_500` (clean collection) but evaluation runs on `nq_poisoned_v4`, the score distributions differ and z-scores are miscalibrated.
**Why it happens:** The clean corpus has different chunk sizes/content than the poisoned corpus.
**How to avoid:** Always compute Signal 4 baseline using the same collection that will be used at eval time (`nq_poisoned_v4`). Use only passages with `passage_id < 20000` (clean) from the canary query results.

### Pitfall 5: FPR Not Measured Without run_eval.py Extension
**What goes wrong:** Phase 3.1 success criterion 6 requires FPR on clean queries. Current `run_eval.py` does not track `chunks_removed` per query — it only tracks `hijacked` flags.
**Why it happens:** FPR measurement requires comparing pre-defense vs. post-defense chunk count on `paired=False` queries.
**How to avoid:** Add `"chunks_removed": n_before - n_after` to each query result dict in `run_eval.py`, and compute `fpr = sum(r["chunks_removed"] > 0 for r in results if not r["paired"]) / n_unpaired` in the aggregate block.

### Pitfall 6: T4 Count Too Small for Standalone Statistics (9 passages, 3 attack sets)
**What goes wrong:** Reporting T4-only signal ablation metrics gives unreliable numbers (n=9 passages across 3 test queries).
**Why it happens:** Phase 2.4 implemented the minimum required 3 attack sets (ROADMAP SC-2).
**How to avoid:** Report T3 and T4 ASR together in the ablation table as "Held-out tiers (T3+T4)" — this gives n=59 test positives. Report T4 co-retrieval rate (9%) separately as a finding, not as a defense signal metric.

### Pitfall 7: LR Feature Normalization Omission
**What goes wrong:** BERT prob is in [0,1], perplexity z-score ranges 0-5+, imperative ratio in [0,1], retrieval z in [-2,4+]. Without scaling, the LR will overweight perplexity and underweight the bounded signals.
**Why it happens:** LR with unequal feature scales assigns weight proportional to the variance of each feature.
**How to avoid:** Fit a `StandardScaler` on training features before fitting LR. Save the scaler to JSON alongside the LR coefs. Transform test features with the same fitted scaler.

## Code Examples

### Save/Load LR Meta-Classifier as JSON (avoids pickle brittleness)

```python
# Source: sklearn 1.8.0 LogisticRegression attributes (VERIFIED available in env)
import json
import numpy as np

def save_lr_to_json(lr, scaler, path: str):
    data = {
        "lr_coef": lr.coef_.tolist(),
        "lr_intercept": lr.intercept_.tolist(),
        "lr_classes": lr.classes_.tolist(),
        "scaler_mean": scaler.mean_.tolist(),
        "scaler_scale": scaler.scale_.tolist(),
    }
    with open(path, "w") as f:
        json.dump(data, f, indent=2)

def load_lr_from_json(path: str):
    from sklearn.linear_model import LogisticRegression
    from sklearn.preprocessing import StandardScaler
    with open(path) as f:
        data = json.load(f)
    lr = LogisticRegression()
    lr.coef_ = np.array(data["lr_coef"])
    lr.intercept_ = np.array(data["lr_intercept"])
    lr.classes_ = np.array(data["lr_classes"])
    scaler = StandardScaler()
    scaler.mean_ = np.array(data["scaler_mean"])
    scaler.scale_ = np.array(data["scaler_scale"])
    return lr, scaler
```
