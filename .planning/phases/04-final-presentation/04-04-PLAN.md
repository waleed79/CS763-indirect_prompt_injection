    The exact command to run in a Windows Terminal foreground window (Game Bar requires a non-Desktop foreground app per RESEARCH §Pitfall 6):
    ```bash

        --model mistral:7b \
    ```

    `## 4. Capture Procedure`
    Step-by-step:
    1. Open Windows Terminal (the foreground app — Game Bar refuses Desktop / File Explorer)
    2. Press `Win + G` → Game Bar overlay appears
    3. Click the Capture widget → Start recording (or `Win + Alt + R`)
    4. Run the command from Section 3 in the terminal
    5. Wait for both queries to complete (~30-40 seconds total)
    6. Press `Win + Alt + R` again to stop
    7. Output saved to `%USERPROFILE%\Videos\Captures\<timestamp>.mp4`

    `## 5. Post-Process (mp4 → palette-optimized GIF)`
    Verbatim from RESEARCH §Code Examples lines 407-422:
    ```bash
    INPUT="$USERPROFILE/Videos/Captures/<latest>.mp4"
    PALETTE="figures/_palette.png"
    OUT="figures/demo_tier2_mistral.gif"

    # Step 1: generate optimal palette from the source
    "C:/Users/muham/AppData/Local/Microsoft/WinGet/Links/ffmpeg.exe" -y -i "$INPUT" \
        -vf "fps=10,scale=960:-1:flags=lanczos,palettegen" "$PALETTE"

    # Step 2: apply palette (smaller file, better colors, infinite loop per Pitfall 2)
    "C:/Users/muham/AppData/Local/Microsoft/WinGet/Links/ffmpeg.exe" -y -i "$INPUT" -i "$PALETTE" \
        -lavfi "fps=10,scale=960:-1:flags=lanczos [x]; [x][1:v] paletteuse" \
        -loop 0 "$OUT"

    rm "$PALETTE"
    ```
    Target: ≤1 MB. If file >1 MB, drop to `fps=8` or `scale=720:-1`. If file <50 KB, the recording was too short or single-frame — re-record.
